Artists on Twitter Finder
Foire aux Questions

Pourquoi faut-il entrer un lien vers une page web, et non vers une image ?

Car il est nécessaire de trouver le ou les comptes Twitter de l'artiste. En effet :

  • Il est impossible d'indexer toutes les images publiées sur Twitter. De plus, cela construirait une base de données à 99% inutile.
  • Et il est impossible de distinguer les Tweets des artistes de Tweets de personnes qui repostent leurs illustrations.

Ainsi, il faut analyser la source de l'illustration pour trouver le ou les comptes Twitter de l'artiste de l'illustration.

Cette analyse est donc différente pour chaque site source. Il faut donc des algorithmes différent pour chaque site. C'est pour cela qu'il y a une liste restreinte de sites supportés.

Pourquoi les illustrations peuvent être très longues à être traitées ?

Il y a deux limites principales :

  • Les limites du nombre de requêtes sur les API de Twitter. Et elles sont très drastiques.
  • Le temps d'analyse des images.

Ainsi, si il faut indexer ou mettre à jour l'indexation d'un compte Twitter, le temps de traitement de l'illustrations va être très long. Et ce temps augmente avec le nombre d'illustrations en cours de traitement dans le serveur.

Comment les comptes Twitter des artistes sont recherchés ?

La recherche de comptes Twitter est faite par le "Link Finder". Comme son nom l'indique, il recherche des liens. Il va donc rechercher sur la page web de l'illustration, ainsi que sur le profile de l'artiste, si il/elle a indiqué un lien vers son compte Twitter.

Si plusieurs comptes Twitter sont trouvés, ils seront tous considérés comme celui de l'artiste.

Détails de la recherche de comptes Twitter pour chaque site supporté :

  • DeviantArt : Dans la description de l'illustration et dans la biographie de l'artiste sur son profile.
  • Pixiv : Dans les liens indiqués par l'artiste sur son profile.
  • Boorus (Danbooru, Derpibooru, Furbooru) : Dans les informations sur l'artiste, ainsi que sur la source de l'illustration, si elle vient d'un site supporté. Attention : Si la source est un Tweet, le compte Twitter associé n'est pas considéré comme le compte Twitter de l'artiste.

Notes :

  • La recherche de comptes Twitter se fait indépendamment pour chaque illustration. Aucune association n'est stockée.
  • Si les comptes Twitter trouvés sont inexistants ou supprimés, suspendus, désactivés, ou privés, ils seront considéré comme invalides, car aucune indexation ni aucune recherche ne peut être faite.
  • Sur les Boorus, si un liens vers le compte DeviantArt ou Pixivi de l'artiste est trouvé, il sera aussi analysé.
  • Si un lien Linktree est trouvé, cette page Linktree sera aussi analysée.

Je suis un artiste, j'ai un compte Twitter, mais il n'est pas trouvé. Pourquoi ?

Vérifiez que vous avez bien mis un lien vers votre compte Twitter là où le serveur recherche (Voir ci-dessus).

Quelles données sont stockées ?

Les données suivantes sont stockées dans la base de données :

  • Comptes Twitter : ID du compte Twitter, et curseurs d'indexation de ce compte.
  • Tweets : ID du Tweet, ID du compte Twitter associé, et liste des hashtags (Ne servent à rien).
  • Images : ID de l'image, histogramme de cette image, et ID du Tweet associé.

Note : Il est impossible de "reconstruire" les images à partir des histogrammes.
